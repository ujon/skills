# 연구노트 상세 작성 가이드

## 법적 근거

- **국가연구개발혁신법** 제35조
- **국가연구개발혁신법 시행령** 제65조
- **국가연구개발사업 연구노트 지침** (과학기술정보통신부 고시)

### 핵심 의무 사항
- 과학기술 분야 연구개발과제에 참여하는 연구자와 연구개발기관의 장은 연구노트를 작성·관리해야 한다.
- 계획서에 연구원으로 등록된 모든 인원이 개별적으로 작성해야 한다.
- 연구개발기관은 연구노트를 30년간 보관·관리해야 한다.

## 연구노트의 4대 핵심 요건

### 1. 기록 시점의 증명
- 작성 날짜를 명확히 기재
- 전자연구노트의 경우 타임스탬프 자동 기록
- 서면 연구노트의 경우 지워지지 않는 필기구로 날짜 기재

### 2. 기록자의 확인
- 기록자(연구원)의 서명 필수
- 소속, 직위 기재 권장

### 3. 제3자(점검자) 확인
- 해당 연구에 직접 참여하지 않은 제3자의 서명
- 기록 내용이 해당 시점에 존재했음을 객관적으로 증명
- 점검자는 내용을 이해할 수 있는 동일/유사 분야 전문가 권장

### 4. 위·변조 방지
- 수정 시 원문을 삭제하지 않고 취소선 처리
- 수정 사유와 일자 기재
- 빈 공간에 사선 표시하여 추후 기입 방지

## 작성 스타일별 톤 가이드

연구노트의 작성 스타일에 따라 문체와 구성이 달라진다.

### 공통 원칙 (모든 스타일 적용)

1. **연구노트는 '노트'다.** 보고서나 교과서가 아니라, 연구자가 당일 수행한 내용을 기록하는 1차 기록물이다. 지나치게 정제된 문서보다는 연구 현장의 맥락이 살아있는 기록이 좋다.
2. **문장 서술을 기본으로 한다.** bullet point 나열은 데이터형·회의록형 등 구조화가 본질인 스타일에서만 사용한다. 그 외 스타일에서는 문장으로 서술하되, 표·수식 등 구조화가 효과적인 부분만 선별적으로 사용한다.
3. **볼드 소제목 나열을 피한다.** `**정의:** ~이다. **특징:** ~이다.` 패턴은 백과사전 느낌을 준다. 한 문단 안에서 자연스럽게 흐르도록 쓴다.
4. **섹션 간 맥락이 이어져야 한다.** 각 섹션이 독립된 문서처럼 읽히지 않도록, 이전 섹션의 내용이 다음 섹션으로 자연스럽게 연결되어야 한다.
5. **참고자료는 본문에 녹인다.** 별도 링크 목록을 만들지 말고, "~논문에서 ~을 확인하였다", "~자료에 따르면" 형태로 본문 속에 언급한다. 출처가 많으면 말미에 참고문헌 섹션을 두되, 본문에서 최소 한 번은 언급해야 한다.

### 기록 중심 스타일

**서술형 (narrative)** — 시간순 일기체로 연구 과정을 서술한다.
- 문체: 경어체 과거형 ("~수행하였다", "~확인하였다")
- 흐름: 배경 → 수행 → 관찰 → 소감 순으로 자연스럽게 이어짐
- 핵심 원칙:
  - 시간 순서가 드러나야 한다. "먼저 ~하였고", "이후 ~를 확인하였다" 등.
  - 예상과 다른 결과, 당일 느낀 점 등 주관적 관찰도 포함한다.
- 예시: "금일 A 조건에서 실험을 수행하였다. 예상과 달리 B 결과가 관찰되었으며, 이는 C 요인에 의한 것으로 추정된다. 내일은 C 변수를 통제한 상태에서 재실험을 진행할 예정이다."

**프로토콜형 (protocol)** — 제3자가 재현할 수 있도록 절차를 상세 기술한다.
- 문체: 간결한 지시/설명체 ("~한다", "~처리한다")
- 흐름: 목적 → 재료/장비 → 단계별 절차 → 결과
- 핵심 원칙:
  - 절차는 번호 매긴 단계(Step)로 기술한다. 이 스타일에서는 리스트가 적절하다.
  - 모호한 표현을 피한다: "적당량" → "50 mL", "잠시" → "10분간"
- 예시: "Step 1: 시료 50mg을 칭량한다. Step 2: 25°C 항온조에서 30분간 반응시킨다."

**데이터형 (data-driven)** — 수치와 표 중심으로 기록하며 해석은 최소화한다.
- 문체: 명사형 위주, 간결한 메모체
- 흐름: 조건 표 → 데이터 테이블 → 통계 요약
- 핵심 원칙:
  - 표와 수치가 주인공이다. 서술은 표를 보충하는 최소한의 설명만 한다.
  - 단위를 반드시 명시하고, 측정 조건(n수, 반복 횟수)을 기재한다.
- 예시: "측정값: 45.2 MPa (n=3, SD=1.8) / 목표: ≥40 MPa → 달성"

**회의록형 (meeting)** — 회의 내용을 구조화하여 기록한다.
- 문체: 요약체, 항목별 나열
- 흐름: 회의 정보(일시/참석자) → 논의 → 결정 → 후속조치
- 핵심 원칙:
  - 결정 사항과 담당자/기한이 명확해야 한다.
  - 논의 과정은 요약하되, 결정의 근거가 드러나야 한다.
- 예시: "[결정] A 방식으로 추가 실험 진행 / [담당] 박OO / [기한] 3/30"

### 사고 중심 스타일

**문제해결형 (troubleshooting)** — 문제 발생부터 해결까지 논리적으로 추적한다.
- 문체: 분석적 서술체
- 흐름: 문제 현상 → 원인 추정 → 시도한 해결책 → 결과/교훈
- 핵심 원칙:
  - 시도했지만 실패한 접근도 기록한다. "처음에 A를 시도했으나 효과가 없었고, 이후 B를 적용하여 해결하였다."
  - 단순 결과뿐 아니라 교훈("다음에는 ~를 먼저 확인해야겠다")도 남긴다.
- 예시: "센서 값에 drift가 발생하여 원인을 조사하였다. 처음에는 센서 자체 불량을 의심했으나 교체 후에도 동일 현상이 재현되었다. 온도 보상 로직을 확인한 결과 보정식이 미적용된 것을 발견하였고, 적용 후 오차가 2% 이내로 개선되었다."

**의사결정형 (decision)** — 선택의 근거와 맥락을 기록한다.
- 문체: 논증체 ("~이유로 ~을 선택하였다")
- 흐름: 배경/제약 → 검토한 대안 비교 → 선택 근거 → 후속 조치
- 핵심 원칙:
  - 대안 비교 시 표를 활용할 수 있으나, 선택 근거는 반드시 문장으로 서술한다.
  - 나중에 "왜 이 결정을 했는가"를 되짚을 수 있도록 제약 조건과 맥락을 명시한다.
- 예시: "A 방식(비용 높음, 정확도 98%)과 B 방식(비용 낮음, 정확도 95%)을 비교한 결과, 예산 제약을 고려하여 B를 채택하였다. 정확도 3% 차이는 현재 요구사양(±5%) 범위 내이므로 허용 가능하다고 판단하였다."

**학습기록형 (study-log)** — 새로 학습한 내용과 이해 과정을 정리한다.
- 문체: 자기 언어로 재구성한 서술체. "~을 알게 되었다", "~이 흥미로웠다", "~가 아직 와닿지 않는다" 등 학습자의 시점이 드러나야 한다.
- 흐름: 학습 동기 → 이해한 내용을 자기 말로 풀어쓰기 → 생각/의문 → 다음 방향
- 핵심 원칙:
  - **문장으로 서술한다.** 정의·개념을 나열하지 말고, "~을 조사해보니 ~라는 것을 알게 되었다" 식으로 학습 과정 자체를 기록한다.
  - **교과서 복사를 하지 않는다.** 표준이나 정의를 그대로 옮기지 말고, 본인이 이해한 방식으로 풀어쓴다. 볼드 소제목(`**정의:**`)으로 항목을 나누지 않는다.
  - **의문점은 맥락과 함께 쓴다.** "~필요하다"로 끝나는 할일 목록이 아니라, 왜 그것이 궁금한지, 어디서 막혔는지를 서술한다.
  - **섹션 간 흐름이 이어진다.** 이전 섹션의 내용이 다음 섹션의 의문으로, 의문이 학습 계획으로 자연스럽게 연결되어야 한다.
  - **참고자료는 본문에 녹인다.** 별도 링크 목록 대신 "~논문에서 ~을 확인하였다", "~자료에 따르면" 형태로 본문 속에 언급한다.
- 잘된 예시:
  ```
  개방형 제어기(OAC)라는 개념을 처음 접했을 때는 단순히 소스가 공개된
  제어기를 뜻하는 줄 알았는데, 조사해보니 핵심은 '벤더 중립적 인터페이스'에
  있었다. 즉, 서로 다른 제조사의 모듈을 하나의 플랫폼에서 교체할 수 있도록
  인터페이스를 표준화하는 것이 OAC의 본질이다.

  미국(OMAC), 유럽(OSACA), 일본(OSEC)이 각각 표준을 만들었다는 점이
  흥미로웠는데, 이 세 표준이 실제로 서로 호환되는 건지 아니면 각자의 생태계
  안에서만 동작하는 건지가 아직 명확하지 않다. 이 부분은 OMAC API 규격서를
  직접 읽어봐야 판단할 수 있을 것 같다.
  ```
- 피해야 할 패턴:
  - `**개방형 제어기의 정의:** ~이다.` → 볼드 소제목 + 사전적 정의 나열
  - `- ~필요하다. / - ~필요하다.` → 의문점을 bullet 할일 목록으로 처리
  - `- ~분석한다. / - ~평가한다.` → 학습 계획을 task list로 처리

### 관리 중심 스타일

**진척보고형 (progress)** — 계획 대비 실적과 이슈를 보고한다.
- 문체: 보고체, 간결한 현황 기술
- 흐름: 기간 → 계획 vs 실적 → 이슈/대응 → 향후 계획
- 핵심 원칙:
  - 계획 대비 실적을 비교하는 구조이므로, 표를 활용해도 좋다.
  - 단순 진행률뿐 아니라 지연/변경 사유와 대응 방안까지 기록한다.
- 예시: "이번 주 목표는 시제품 v2 완성이었으나, 부품 수급이 2주 지연되어 80% 수준에서 마무리하였다. 대체 부품으로 테스트를 진행 중이며, 다음 주까지 호환성 검증을 완료할 예정이다."

**문헌정리형 (literature)** — 논문/자료를 체계적으로 정리하고 연구와의 연관성을 분석한다.
- 문체: 학술적 요약체이되, 본인의 해석과 판단이 포함되어야 한다
- 흐름: 문헌 서지 정보 → 핵심 발견 → 본 연구 관련성 → 적용 방안
- 핵심 원칙:
  - 논문 내용을 그대로 옮기지 않고, 본 과제와의 관련성 관점에서 요약한다.
  - "이 논문에서 주목할 점은 ~이다", "본 과제에 적용한다면 ~할 수 있을 것이다" 등 연구자의 시각이 드러나야 한다.
- 예시: "Kim et al.(2025)은 A 소재의 B 특성을 다양한 온도 조건에서 규명하였다. 특히 60°C 이상에서 나타나는 상전이 거동은 본 과제의 C 공정에서도 유사하게 발생할 가능성이 있어, 향후 실험 설계 시 온도 범위를 확장하여 검증할 필요가 있다."

## 섹션별 작성 요령 (프로토콜형/데이터형 기본)

### 연구 목적 / 배경
작성 요령:
- 해당 연구/실험을 수행하게 된 배경과 필요성을 2~3문단으로 기술
- R&D 계획서 상의 세부 과제 또는 마일스톤과의 연계성을 명시
- 이전 실험/연구에서 도출된 과제가 있으면 언급
- 구체적인 연구 가설이나 검증 목표를 명시

예시 문구:
"본 연구는 [과제명]의 2차년도 세부과제 '[세부과제명]'과 관련하여, 
[구체적 기술/현상]의 [특성/성능]을 규명하기 위해 수행되었다.
선행 연구(연구노트 No.XX, 20XX.XX.XX)에서 확인된 [문제점/과제]를 
해결하기 위해 [접근 방법]을 적용하였다."

### 연구 내용 및 방법
작성 요령:
- 제3자가 동일한 실험/연구를 재현할 수 있도록 구체적으로 기술
- 사용 장비는 모델명, 제조사, 주요 설정값까지 기재
- 시약/재료는 제조사, 순도/등급, lot번호(가능한 경우)까지 기재
- 실험 절차를 시간순 또는 단계순으로 번호를 매겨 기술
- 모호한 표현 지양: "적당량" → "50 mL", "잠시" → "10분간"

예시 구조:
```
[사용 장비 및 재료]
- 분석 장비: XX사 모델 YY (S/N: ZZZZ)
- 시약: AA (BB사, 순도 99.5%, Cat. No. CCCC)
- 소프트웨어: DD v3.2 (EE Inc.)

[실험 절차]
1) 시료 XX를 YY 조건에서 ZZ분간 전처리
2) AA 용액 50 mL에 시료를 투입하고 BB°C에서 CC시간 반응
3) 반응 후 DD 방법으로 생성물 분리
4) EE 장비를 이용하여 FF 조건에서 분석 수행
```

### 결과
작성 요령:
- 측정 데이터를 표 또는 목록 형태로 정리
- 성공 결과뿐 아니라 실패 결과도 반드시 기록
- 데이터의 단위를 명확히 표시
- 그래프나 이미지가 있는 경우 "첨부 [번호] 참조"로 연결
- 반복 실험의 경우 각 시행의 결과를 모두 기록
- 정성적 관찰 사항도 기록 (색 변화, 이상 현상 등)

예시 구조:
```
[실험 결과]
- 시료 A: 강도 45.2 MPa (목표값 40 MPa 이상 → 달성)
- 시료 B: 강도 38.7 MPa (목표값 미달)
- 시료 C: 측정 중 장비 오류 발생, 재측정 필요

[관찰 사항]
- 시료 B에서 반응 온도 상승 시(80°C 이상) 변색 현상 관찰
- 해당 현상은 선행 문헌에서 보고되지 않은 사항으로, 추가 분석 필요
```

### 고찰 및 결론
작성 요령:
- 실험 결과를 연구 목적과 연계하여 해석
- 예상과 일치하는 결과와 그렇지 않은 결과를 구분하여 분석
- 실패한 경우 원인 분석과 가능한 해결 방안 제시
- 향후 연구 방향이나 후속 실험 계획을 기술
- 이 결과가 전체 과제 진행에 미치는 영향 언급

예시 문구:
"시료 A의 결과는 목표 성능을 달성하여, [공정/방법]의 유효성이 확인되었다.
반면, 시료 B는 목표에 미달하였으며, 이는 [원인 추정]에 기인한 것으로 판단된다.
향후 [조건 변경]을 통해 개선 여부를 확인할 예정이다."

### 특이사항 / 비고
기록 대상:
- 장비 고장, 캘리브레이션 이슈
- 실험 환경 변화 (온도, 습도, 정전 등)
- 연구 계획 대비 변경 사항과 변경 사유
- 외부 기관 의뢰 분석 내역
- 안전 관련 사고나 유의 사항

## 과제 평가와의 연관성

### 과정평가 지표
2022년부터 R&D 최종 평가에 성과와 과정평가를 동시 진행한다. 
과정평가 중 '연구 수행 방법 및 과정의 적절성' 지표에서 연구노트가 핵심 증빙이 된다.

### 성실실패 인정 요건
과제 수행 중 목표 미달성 시에도, 연구노트가 성실하게 작성되었다면 
'성실 실패'로 인정받아 과제비 환수 및 참여 제한 등의 제재를 면할 수 있다.

### 불성실 수행 판정 위험
연구노트가 부실하거나 없는 경우:
- 불성실 수행 판정
- 정부지원금 환수
- 향후 과제 참여 제한
